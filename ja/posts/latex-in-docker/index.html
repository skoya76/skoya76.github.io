Docker, VSCode, Devcontainer を使って LaTeX 環境を構築する

10月 5, 2024·
塩崎 功也
· 3 分で読める

本記事では、Docker と VSCode の Devcontainer 機能を利用し、仮想環境上で LaTeX 執筆環境を構築する手順を解説します。Docker コンテナを用いることで、ホスト環境を汚さずに LaTeX のフル機能を利用することができる設定を行うことを主な目的としています。

1. プロジェクトディレクトリの作成

まず、作業用のプロジェクトディレクトリを作成し、その中に .devcontainer ディレクトリを準備します。

mkdir latex-dev-container
cd latex-dev-container
mkdir .devcontainer

2. Dev Container 設定ファイルの作成

.devcontainer ディレクトリ内に devcontainer.json を作成し、以下の設定を記述します。

{
  "name": "LaTeX Dev Container",
  "build": {
    "dockerfile": "Dockerfile",
    "context": ".."
  },
  "workspaceFolder": "/workspace",
  "settings": {
    "terminal.integrated.shell.linux": "/bin/bash",
    "latex-workshop.latex.autoBuild.run": "onSave"
  },
  "extensions": [
    "James-Yu.latex-workshop",
    "ms-vscode-remote.remote-containers"
  ],
  "postCreateCommand": "latexmk -C",
  "remoteUser": "vscode"
}

各設定項目の説明

  • dockerfile: 使用する Dockerfile のパスを指定します（.devcontainer ディレクトリ内の Dockerfile）。
  • context: Docker ビルドのコンテキストを指定します（.. は親ディレクトリを指します）。
  • workspaceFolder: Dev Container 内での作業ディレクトリを指定します（/workspace）。
  • settings: VSCode の設定で、LaTeX Workshop 拡張機能の自動ビルド設定を行います。
    • latex-workshop.latex.autoBuild.run: LaTeX ファイルを保存した際に自動ビルドを実行します。
  • extensions: 使用する VSCode 拡張機能を指定します。
    • James-Yu.latex-workshop: LaTeX Workshop 拡張機能。
    • ms-vscode-remote.remote-containers: Remote - Containers 拡張機能。
  • postCreateCommand: コンテナ作成後に自動的に実行されるコマンド（latexmk のクリーンアップコマンドを実行）。
  • remoteUser: コンテナ内で使用するデフォルトユーザーを vscode に設定します。

3. Dockerfile の作成

次に、以下の内容を .devcontainer/Dockerfile として保存します。

FROM debian:bookworm-slim

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && \    apt-get install -y texlive-full latexmk vim git curl && \    apt-get clean

RUN useradd -m vscode

WORKDIR /workspace

各設定項目の説明

  • debian:bookworm-slim: 最新の Debian ベースの軽量イメージを使用します。
  • パッケージのインストール: LaTeX 関連のパッケージ（texlive-full, latexmk, vim, git, curl など）をインストールします。
  • ユーザー作成: vscode ユーザーを作成し、デフォルトの作業ユーザーとして設定します。
  • 作業ディレクトリ: /workspace を作業ディレクトリに指定します。

4. Dev Container の起動

  1. VSCode でプロジェクトディレクトリを開き、左下の緑色のボタン（Remote Explorer）をクリックします。
  2. 「Reopen in Container」を選択し、コンテナのビルドと起動を待ちます。

5. LaTeX ファイルの作成と PDF の生成

  1. コンテナが起動したら、新しいファイル main.tex を作成し、以下のサンプル LaTeX コードを貼り付けます。

    \documentclass{article}
\begin{document}
\title{LaTeX Dev Container Example}
\author{Kohya Shiozaki}
\date{\today}
\maketitle

\section{Introduction}
This document is compiled inside a Docker Dev Container using VSCode.

\end{document}

  2. ファイルを保存（Ctrl+S）すると、LaTeX Workshop が自動的に latexmk を実行し、PDF ファイル main.pdf が生成されます。

  3. 生成された PDF をプレビューするには、左側の LaTeX Workshop のサイドバーにある「View LaTeX PDF」を選択します。

6. 不要な一時ファイルのクリーンアップ

latexmk によって生成された一時ファイル（.aux, .log, .fls など）が不要な場合は、以下のコマンドをコンテナ内で実行してクリーンアップします。

latexmk -c

これにより、ビルド時に生成された一時ファイルを削除し、ディレクトリをクリーンな状態に保つことができます。

7. .gitignore への追加

バージョン管理システムで .fls などの一時ファイルを無視したい場合は、.gitignore に以下を追加します。

*.fls
*.aux
*.log
*.bbl
*.blg

これにより、これらの一時ファイルが Git などに含まれなくなり、管理がしやすくなります。

これで、VSCode の Dev Container と LaTeX Workshop を使用した LaTeX 執筆環境の設定が完了です。以上の設定を .md ファイルとして保存し、ブログ記事やドキュメントとしてご活用ください。
